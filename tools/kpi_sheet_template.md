# FusionCore KPI Sheet

Create this as a Google Sheet (or local spreadsheet). One row per week, filled every Sunday in under 10 minutes.

## Columns

| Column | What to measure | Where to get it |
|---|---|---|
| Week | YYYY-MM-DD (Monday) | |
| GitHub Stars | Total stars on the repo | github.com/manankharwar/fusioncore - right sidebar |
| GitHub Clones (unique) | Unique cloners this week | Insights > Traffic > Clones |
| GitHub Forks | Total forks | repo sidebar |
| Docs Impressions | Search Console impressions this week | Google Search Console > Performance |
| Docs Clicks | Search Console clicks this week | Google Search Console > Performance |
| Top docs page | Highest traffic page this week | Search Console > Pages |
| Qualified conversations | # of people who asked a real question about their use case (GitHub issues, email, forum) | count manually |
| Proposals sent | # of fixed-price proposals sent this week | count manually |
| Active negotiations | # of proposals in back-and-forth | count manually |
| MRR (USD) | Monthly recurring revenue | sum of active support agreements |
| One-time revenue (USD) | Fixed-price integration payments received this week | |
| Notes | Anything notable that happened | |

## Weekly fill process (10 min max)

1. Open GitHub repo > Insights > Traffic: copy Clones (unique this week)
2. Open Google Search Console: copy Impressions and Clicks for the last 7 days
3. Count any new qualified conversations since last Sunday
4. Count proposals sent and active
5. Note any revenue received
6. Write one sentence in Notes about what happened
